### 3단계: 로컬 단독 버전 기능 개선 (GoldERP-v5.html)

#### 완료된 기능 수정들

**3-1. 발주관리 상태 변경 개선**
- '발주완료' 상태 완전 제거
- 상태변경 → 입고완료만 선택 가능
- 체크박스 선택 시 입고완료로 일괄 변경 기능

**3-2. 발주관리 → 출고관리 연동**
- 입고완료 시 자동으로 출고관리 페이지로 이동
- 고객발주: 연결된 주문 stage를 'C'(출고대기)로 변경
- 자체발주: 출고관리에 보라색 [자체] 뱃지로 표시

**3-3. 출고관리 체크박스 일괄 출고 + 재고 자동 등록**
- 출고 대기 항목에 체크박스 추가
- 체크 후 "출고완료 → 재고등록" 버튼으로 일괄 처리
- 재고관리 DB에 자동 등록 후 재고관리 페이지로 이동

**3-4. 재고관리 탭 구분**
- 📋 전체 | 🏪 진열재고 | 🔒 금고재고 | 📦 주문재고 탭 추가
- 각 탭별 건수 표시
- 신규 등록 시 현재 탭에 맞는 보관위치 자동 선택

**3-5. 출고완료 상태 유지 버그 수정**
- `syncOrderStageFromPurchases()`가 stage를 'P'로 리셋하던 문제 해결
- stage 'C'(출고대기), 'F'(출고완료)는 덮어쓰지 않도록 수정

**3-6. 발주번호 표시 추가**
- 출고관리 테이블에 발주번호 열 추가
- 재고관리 테이블에 발주번호 열 추가
- 발주관리 테이블에 주문번호 열 추가

**3-7. 주문등록 카탈로그 패널**
- 새 주문 입력 시 상품명에서 카테고리 탭으로 제품 선택
- 한 글자만 입력해도 카탈로그 자동 표시 (검색 필터링)
- 상품 이미지 포함, 선택 시 상품명·거래처·금액 자동 입력

**3-8. 바코드 입고 시스템 (방안 A) — ✅ 완료**
- 발주관리 각 행에 🏷️ 바코드 인쇄 버튼 추가 완료
- 사이드바에 📡 바코드 입고 메뉴 추가 완료
- 바코드 스캔 페이지 HTML 추가 완료 (좌측 스캔 영역 + 우측 오늘 입고내역 + 하단 대기목록)
- 핵심 JS 함수 구현 완료:
  - `printPurchaseBarcode(idx)` — 발주별 고유 바코드 생성 및 라벨 인쇄
  - `processBarcodeIncoming(code)` — 바코드 스캔 → 입고처리 → 재고 자동등록
  - `setBarcodeLocation(loc)` — 보관위치(진열/금고/주문) 선택
  - `renderBarcodeIncoming()` — 바코드 입고 페이지 전체 렌더링
  - `renderBarcodeIncomingList()` — 오늘 입고 내역 표시
  - `renderBarcodePendingTable()` — 입고 대기 발주 목록 표시
- 전체 흐름 테스트 완료: 바코드 인쇄 → 스캔 → 입고완료 → 주문연동(stage C) → 재고 자동등록
- 중복 스캔 방지, 미등록 바코드 에러 처리 확인 완료

## 바코드 입고 시스템 사용법 (완성)

### 1단계: 바코드 발행
- **발주관리** 페이지에서 각 발주 행의 **🏷️ 버튼** 클릭
- 고유 바코드(PO-PO001-XXXXXX 형식)가 자동 생성됨
- 인쇄 버튼으로 라벨 출력 가능

### 2단계: 바코드 스캔 입고
- 사이드바 **📦 재고 > 📡 바코드 입고** 메뉴 진입
- 보관위치(🏪 진열 / 🔒 금고 / 📦 주문) 먼저 선택
- 스캔 입력창에 바코드 스캔 또는 직접 입력 후 Enter
- 자동으로: 발주상태 → 입고완료, 주문 stage → C, 재고에 자동 등록

### 3단계: 확인
- 오늘 입고 내역이 우측에 실시간 표시
- 하단에 아직 미입고인 발주 목록 표시
- 재고관리에서 입고된 항목 확인 가능
